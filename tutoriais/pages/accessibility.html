♿ Acessibilidade Web: WCAG e Práticas Inclusivas

📖 20 min de leitura 🎯 Nível: Iniciante/Intermediário 💻 HTML Semântico • ARIA 🌍 Inclusão Digital

📌 Introdução

Acessibilidade web não é apenas uma boa prática - é um direito. Milhões de pessoas com deficiências visuais, motoras, auditivas ou cognitivas dependem de sites acessíveis para participar da sociedade digital. As diretrizes WCAG (Web Content Accessibility Guidelines) fornecem um padrão internacional para tornar o conteúdo web mais acessível.

Neste tutorial, vamos aprender os princípios fundamentais da acessibilidade e como implementá-los em seus projetos.

🎯 Objetivos

  • Entender os 4 princípios do WCAG (Perceptível, Operável, Compreensível, Robusto)
  • Aprender a usar HTML semântico corretamente
  • Implementar atributos ARIA quando necessário
  • Garantir contraste e legibilidade adequados
  • Tornar componentes interativos acessíveis por teclado

⚠️ Erro Comum #1: Usar divs para tudo

❌ Erro: Estrutura semântica pobre 
<div class="header">
    <div class="nav">
        <div class="nav-item">Home</div>
        <div class="nav-item">Sobre</div>
    </div>
</div>
<div class="main">
    <div class="article">
        <div class="title">Título</div>
        <div class="content">Conteúdo...</div>
    </div>
</div>
<div class="footer">Rodapé</div>

Problema: Leitores de tela não conseguem entender a estrutura. Navegação por teclado é limitada.

✅ Correção: Usar HTML semântico 
<header>
    <nav>
        <ul>
            <li><a href="/">Home</a></li>
            <li><a href="/sobre">Sobre</a></li>
        </ul>
    </nav>
</header>
<main>
    <article>
        <h1>Título</h1>
        <p>Conteúdo...</p>
    </article>
</main>
<footer>Rodapé</footer>

⚠️ Erro Comum #2: Contraste de cores insuficiente

❌ Erro: Cores com baixo contraste 
/* Contraste insuficiente (ratio ~2.5:1) */
body {
    background-color: #ffffff;
    color: #cccccc;  /* Cinza claro */
}

.button {
    background-color: #f0f0f0;
    color: #a0a0a0;
}

Problema: Texto difícil de ler para pessoas com baixa visão ou daltonismo.

✅ Correção: Garantir contraste mínimo (4.5:1 para texto normal) 
/* Contraste adequado (ratio > 4.5:1) */
body {
    background-color: #ffffff;
    color: #1a1a1a;  /* Preto/cinza escuro */
}

.button {
    background-color: #0066cc;  /* Azul escuro */
    color: #ffffff;  /* Branco */
}

/* Ferramentas para verificar contraste:
   - Chrome DevTools
   - WebAIM Contrast Checker
   - Contrast Ratio (a11y)
📊 Padrões WCAG:

Nível AA: contraste 4.5:1 para texto normal, 3:1 para texto grande
Nível AAA: contraste 7:1 para texto normal, 4.5:1 para texto grande

⚠️ Erro Comum #3: Elementos interativos sem foco visível

❌ Erro: Remover outline sem alternativa 
/* Isso é um ERRO GRAVE! */
:focus {
    outline: none;
}

/* Ou pior - esconder o foco sem alternativa */
button:focus {
    outline: none;
    opacity: 0.8;
}

Problema: Usuários de teclado perdem a noção de onde estão na página.

✅ Correção: Manter ou estilizar o foco adequadamente 
/* Opção 1: Manter outline padrão */
:focus {
    outline: 2px solid #0066cc;
    outline-offset: 2px;
}

/* Opção 2: Estilizar com box-shadow */
:focus-visible {
    outline: none;
    box-shadow: 0 0 0 3px rgba(0, 102, 204, 0.4);
    border-radius: 4px;
}

/* Opção 3: Para elementos específicos */
button:focus-visible {
    outline: none;
    background-color: #004999;
    transform: scale(1.02);
}

/* Nunca remova o foco sem uma alternativa visual clara! */

⚠️ Erro Comum #4: Falta de labels em formulários

❌ Erro: Placeholder como único label 
<input type="text" placeholder="Digite seu nome">
<input type="email" placeholder="Email">

<div class="checkbox">
    <input type="checkbox"> Aceito termos
</div>

Problema: Leitores de tela não associam labels aos campos. Placeholder desaparece ao digitar.

✅ Correção: Usar labels associadas 
<label for="nome">Nome completo:</label>
<input type="text" id="nome" name="nome" required>

<label for="email">E-mail:</label>
<input type="email" id="email" name="email">

<div class="checkbox">
    <input type="checkbox" id="termos" name="termos">
    <label for="termos">Aceito os termos e condições</label>
</div>

<fieldset>
    <legend>Preferência de contato:</legend>
    <input type="radio" id="email_contato" name="contato" value="email">
    <label for="email_contato">E-mail</label>
    <input type="radio" id="telefone" name="contato" value="telefone">
    <label for="telefone">Telefone</label>
</fieldset>

🚀 Evolução: Componentes acessíveis

1. Modal/Dialog Acessível

<!-- HTML -->
<button id="openModal" aria-haspopup="dialog">Abrir Modal</button>

<div id="modal" class="modal" role="dialog" aria-modal="true" aria-labelledby="modalTitle" hidden>
    <div class="modal-content">
        <h2 id="modalTitle">Título do Modal</h2>
        <p>Conteúdo do modal...</p>
        <button id="closeModal">Fechar</button>
    </div>
</div>

<script>
const modal = document.getElementById('modal');
const openBtn = document.getElementById('openModal');
const closeBtn = document.getElementById('closeModal');

function openModal() {
    modal.hidden = false;
    // Foco vai para o modal
    modal.querySelector('h2').focus();
    // Armazenar elemento que abriu
    openBtn.setAttribute('aria-expanded', 'true');
    // Travar scroll
    document.body.style.overflow = 'hidden';
}

function closeModal() {
    modal.hidden = true;
    openBtn.focus();
    openBtn.setAttribute('aria-expanded', 'false');
    document.body.style.overflow = '';
}

// Fechar com ESC
document.addEventListener('keydown', (e) => {
    if (e.key === 'Escape' && !modal.hidden) {
        closeModal();
    }
});

openBtn.addEventListener('click', openModal);
closeBtn.addEventListener('click', closeModal);
</script>

2. Menu Dropdown Acessível

<nav>
    <button id="menuButton" 
            aria-haspopup="true" 
            aria-expanded="false"
            aria-controls="dropdownMenu">
        Menu
    </button>
    <ul id="dropdownMenu" 
        role="menu" 
        aria-labelledby="menuButton"
        hidden>
        <li role="menuitem"><a href="#">Opção 1</a></li>
        <li role="menuitem"><a href="#">Opção 2</a></li>
        <li role="menuitem"><a href="#">Opção 3</a></li>
    </ul>
</nav>

<script>
const menuButton = document.getElementById('menuButton');
const dropdown = document.getElementById('dropdownMenu');
let isOpen = false;

function toggleMenu() {
    isOpen = !isOpen;
    dropdown.hidden = !isOpen;
    menuButton.setAttribute('aria-expanded', isOpen);
    
    if (isOpen) {
        const firstItem = dropdown.querySelector('a');
        firstItem.focus();
    }
}

// Navegação por teclado
function handleKeyDown(e) {
    const items = Array.from(dropdown.querySelectorAll('a'));
    const currentIndex = items.indexOf(document.activeElement);
    
    switch(e.key) {
        case 'ArrowDown':
            e.preventDefault();
            const nextIndex = (currentIndex + 1) % items.length;
            items[nextIndex].focus();
            break;
        case 'ArrowUp':
            e.preventDefault();
            const prevIndex = (currentIndex - 1 + items.length) % items.length;
            items[prevIndex].focus();
            break;
        case 'Escape':
            toggleMenu();
            menuButton.focus();
            break;
    }
}

menuButton.addEventListener('click', toggleMenu);
dropdown.addEventListener('keydown', handleKeyDown);
</script>
🎯 Evolução:

Comece com HTML semântico, depois adicione ARIA quando necessário, e evolua para componentes interativos totalmente acessíveis com suporte a teclado e leitores de tela.

📋 Checklist de Acessibilidade

  • ✅ [ ] Usar HTML semântico (header, nav, main, article, etc)
  • ✅ [ ] Todos os inputs têm labels associados
  • ✅ [ ] Contraste de cores ≥ 4.5:1 (texto normal)
  • ✅ [ ] Foco visível em todos os elementos interativos
  • ✅ [ ] Navegação por teclado funcional (Tab, Enter, Esc)
  • ✅ [ ] Imagens têm alt text descritivo
  • ✅ [ ] Vídeos têm legendas ou transcrição
  • ✅ [ ] Páginas têm títulos descritivos (<title>)
  • ✅ [ ] Headings usados em ordem hierárquica (h1, h2, h3...)
  • ✅ [ ] Links têm texto descritivo (não "clique aqui")
  • ✅ [ ] ARIA usado apenas quando necessário
  • ✅ [ ] Testado com leitor de tela (NVDA, VoiceOver)

💡 Dicas Avançadas

  • Live Regions: Use aria-live="polite" para anúncios dinâmicos
  • Skip Links: Adicione link "Pular para conteúdo" no topo da página
  • Reduzir movimento: Respeite prefers-reduced-motion
  • Ferramentas de teste: Lighthouse, axe DevTools, WAVE

Exemplo: Skip Link

<body>
    <a href="#main-content" class="skip-link">Pular para conteúdo principal</a>
    <header>...</header>
    <main id="main-content" tabindex="-1">...</main>
</body>

<style>
.skip-link {
    position: absolute;
    top: -40px;
    left: 0;
    background: #0066cc;
    color: white;
    padding: 8px;
    z-index: 100;
}

.skip-link:focus {
    top: 0;
}
</style>

Respeitando prefers-reduced-motion

@media (prefers-reduced-motion: reduce) {
    *,
    *::before,
    *::after {
        animation-duration: 0.01ms !important;
        animation-iteration-count: 1 !important;
        transition-duration: 0.01ms !important;
        scroll-behavior: auto !important;
    }
}

📝 Exercício Prático

Transforme um site não acessível em um site acessível:

  • Pegue um componente comum (carrossel, modal, formulário) que não é acessível
  • Adicione HTML semântico apropriado
  • Implemente navegação por teclado completa
  • Adicione atributos ARIA necessários
  • Teste com leitor de tela (NVDA no Windows, VoiceOver no Mac)
  • Verifique contraste e foco visível
🎯 Desafio extra: Implemente um componente de "Dark Mode" que também respeita prefers-color-scheme e mantenha contraste adequado em ambos os temas.
