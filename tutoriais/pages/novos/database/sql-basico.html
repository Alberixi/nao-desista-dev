🗄️ SQL: Consultas Essenciais

📖 30 min de leitura 🎯 Nível: Iniciante 💻 Banco de Dados

📌 Introdução

SQL (Structured Query Language) é a linguagem padrão para manipulação de bancos de dados relacionais. Dominar SQL é essencial para qualquer desenvolvedor, seja front-end, back-end ou full-stack. Neste tutorial, vamos aprender as consultas mais importantes do dia a dia.

📊 Estrutura de Exemplo

Vamos usar um banco de dados de exemplo com as tabelas:

-- Tabela USUARIOS
CREATE TABLE usuarios (
    id INT PRIMARY KEY,
    nome VARCHAR(100),
    email VARCHAR(100),
    idade INT,
    cidade VARCHAR(50)
);

-- Tabela PEDIDOS
CREATE TABLE pedidos (
    id INT PRIMARY KEY,
    usuario_id INT,
    produto VARCHAR(100),
    valor DECIMAL(10,2),
    data_pedido DATE,
    FOREIGN KEY (usuario_id) REFERENCES usuarios(id)
);

⚠️ Erro Comum #1: SELECT * em produção

❌ Erro: Usar SELECT * em consultas 
-- Busca todas as colunas, mesmo as desnecessárias
SELECT * FROM usuarios WHERE cidade = 'São Paulo';

Problema: Consome mais memória, rede e processamento, especialmente em tabelas grandes.

✅ Correção: Especificar apenas as colunas necessárias 
-- Busca apenas as colunas que realmente precisa
SELECT id, nome, email FROM usuarios WHERE cidade = 'São Paulo';

⚠️ Erro Comum #2: JOIN sem condição (produto cartesiano)

❌ Erro: JOIN sem ON 
-- Resultado: 3 usuários × 3 pedidos = 9 linhas (produto cartesiano)
SELECT * FROM usuarios, pedidos;

Problema: Retorna combinações inválidas e consome recursos desnecessários.

✅ Correção: Usar JOIN com condição 
-- Retorna apenas combinações válidas
SELECT u.nome, p.produto, p.valor
FROM usuarios u
INNER JOIN pedidos p ON u.id = p.usuario_id;

⚠️ Erro Comum #3: Esquecer GROUP BY com funções agregadas

❌ Erro: Usar COUNT sem GROUP BY 
-- Erro: coluna não agregada sem GROUP BY
SELECT cidade, COUNT(*) FROM usuarios;

Problema: O banco de dados não sabe como agrupar os resultados.

✅ Correção: Adicionar GROUP BY 
-- Agrupa por cidade e conta usuários por cidade
SELECT cidade, COUNT(*) as total_usuarios
FROM usuarios
GROUP BY cidade
ORDER BY total_usuarios DESC;

🚀 Consultas Essenciais

1. SELECT Básico

-- Selecionar colunas específicas
SELECT nome, email FROM usuarios;

-- Com alias (apelidos)
SELECT nome AS "Nome Completo", email FROM usuarios;

2. WHERE (Filtros)

-- Operadores de comparação
SELECT * FROM usuarios WHERE idade >= 25;

-- Operadores lógicos
SELECT * FROM usuarios WHERE cidade = 'São Paulo' AND idade > 20;

-- LIKE (busca parcial)
SELECT * FROM usuarios WHERE nome LIKE 'J%'; -- Começa com J

-- IN (lista de valores)
SELECT * FROM usuarios WHERE cidade IN ('São Paulo', 'Rio de Janeiro');

-- BETWEEN (intervalo)
SELECT * FROM usuarios WHERE idade BETWEEN 20 AND 30;

3. JOINs (Relacionamentos)

-- INNER JOIN: apenas registros que existem em ambas tabelas
SELECT u.nome, p.produto, p.valor
FROM usuarios u
INNER JOIN pedidos p ON u.id = p.usuario_id;

-- LEFT JOIN: todos os usuários, mesmo sem pedidos
SELECT u.nome, COUNT(p.id) as total_pedidos
FROM usuarios u
LEFT JOIN pedidos p ON u.id = p.usuario_id
GROUP BY u.nome;

4. Funções Agregadas

-- COUNT: conta registros
SELECT COUNT(*) as total_usuarios FROM usuarios;

-- SUM: soma valores
SELECT SUM(valor) as total_vendas FROM pedidos;

-- AVG: média
SELECT AVG(idade) as idade_media FROM usuarios;

-- MAX / MIN: máximo e mínimo
SELECT MAX(valor) as maior_pedido FROM pedidos;

5. Ordenação e Limite

-- ORDER BY: ordenar resultados
SELECT * FROM usuarios ORDER BY nome ASC;  -- A-Z
SELECT * FROM pedidos ORDER BY valor DESC; -- Maior para menor

-- LIMIT: limitar resultados
SELECT * FROM usuarios LIMIT 5;

-- Paginação: OFFSET + LIMIT
SELECT * FROM usuarios LIMIT 10 OFFSET 20; -- Página 3 com 10 itens
🎯 Dica de Performance:

Sempre use índices em colunas usadas em WHERE, JOIN e ORDER BY para consultas mais rápidas.

📝 Exercício Prático

Escreva consultas SQL para:

  • Listar os 5 produtos mais caros
  • Calcular o valor total de pedidos por usuário
  • Encontrar usuários que nunca fizeram pedidos
  • Calcular a média de idade por cidade
  • Listar pedidos dos últimos 30 dias
🎯 Soluções: 
-- 1. Top 5 produtos mais caros
SELECT produto, valor FROM pedidos ORDER BY valor DESC LIMIT 5;

-- 2. Valor total por usuário
SELECT u.nome, SUM(p.valor) as total_gasto
FROM usuarios u
LEFT JOIN pedidos p ON u.id = p.usuario_id
GROUP BY u.nome;

-- 3. Usuários sem pedidos
SELECT u.* FROM usuarios u
LEFT JOIN pedidos p ON u.id = p.usuario_id
WHERE p.id IS NULL;

-- 4. Média de idade por cidade
SELECT cidade, AVG(idade) as idade_media
FROM usuarios
GROUP BY cidade;

-- 5. Pedidos últimos 30 dias
SELECT * FROM pedidos
WHERE data_pedido >= CURRENT_DATE - INTERVAL 30 DAY;
