🐳 Docker: Containerização na Prática

📖 35 min de leitura 🎯 Nível: Intermediário 💻 DevOps

📌 Introdução

Docker é a tecnologia de containerização mais popular do mundo. Com ele, você pode empacotar suas aplicações com todas as dependências em containers isolados, garantindo que funcionem da mesma forma em qualquer ambiente. Neste tutorial, vamos aprender desde a instalação até a criação de Dockerfiles e docker-compose.

⚠️ Erro Comum #1: Container com nome já existente

❌ Erro: Tentar criar container com nome em uso 
docker run --name meu-app node:18
docker run --name meu-app node:18
# Error: Conflict. The container name "/meu-app" is already in use

Problema: O Docker não permite nomes duplicados.

✅ Correções possíveis: 
# Opção 1: Remover container existente
docker rm meu-app
docker run --name meu-app node:18

# Opção 2: Usar nome diferente
docker run --name meu-app-2 node:18

# Opção 3: Usar --rm (remove automaticamente ao parar)
docker run --rm --name meu-app node:18

⚠️ Erro Comum #2: Dockerfile ineficiente

❌ Erro: Camadas desnecessárias e ordem incorreta 
FROM node:18
COPY . /app
WORKDIR /app
RUN npm install
CMD ["npm", "start"]

Problema: Cada mudança no código força o re-download de todas as dependências.

✅ Correção: Otimizar camadas do Dockerfile 
FROM node:18-alpine AS builder
WORKDIR /app
# Primeiro copia apenas package.json (aproveita cache)
COPY package*.json ./
RUN npm ci --only=production

# Depois copia o código
COPY . .
EXPOSE 3000
CMD ["node", "server.js"]

🚀 Comandos Essenciais do Docker

1. Gerenciamento de Containers

# Listar containers em execução
docker ps

# Listar todos os containers (incluindo parados)
docker ps -a

# Executar um container
docker run -d --name meu-app -p 3000:3000 node:18

# Parar um container
docker stop meu-app

# Iniciar um container parado
docker start meu-app

# Remover um container
docker rm meu-app

# Ver logs
docker logs meu-app -f

2. Gerenciamento de Imagens

# Listar imagens
docker images

# Baixar imagem
docker pull node:18-alpine

# Construir imagem a partir de Dockerfile
docker build -t minha-app:1.0 .

# Remover imagem
docker rmi minha-app:1.0

3. Dockerfile Básico para Node.js

# Dockerfile
FROM node:18-alpine

# Define diretório de trabalho
WORKDIR /app

# Copia arquivos de dependência
COPY package*.json ./

# Instala dependências
RUN npm ci --only=production

# Copia código fonte
COPY . .

# Expõe a porta
EXPOSE 3000

# Comando de inicialização
CMD ["node", "server.js"]
🎯 Dockerfile Otimizado (Multi-stage build): 
# Stage 1: Build
FROM node:18-alpine AS builder
WORKDIR /app
COPY package*.json ./
RUN npm ci
COPY . .
RUN npm run build

# Stage 2: Produção (imagem menor)
FROM node:18-alpine
WORKDIR /app
COPY --from=builder /app/dist ./dist
COPY --from=builder /app/package*.json ./
RUN npm ci --only=production
EXPOSE 3000
CMD ["node", "dist/server.js"]

🐳 Docker Compose

Para aplicações com múltiplos serviços (API + Banco de Dados), use docker-compose:

# docker-compose.yml
version: '3.8'

services:
  app:
    build: .
    ports:
      - "3000:3000"
    environment:
      - DB_HOST=postgres
      - DB_USER=user
      - DB_PASSWORD=password
    depends_on:
      - postgres
    volumes:
      - ./src:/app/src

  postgres:
    image: postgres:15-alpine
    environment:
      - POSTGRES_USER=user
      - POSTGRES_PASSWORD=password
      - POSTGRES_DB=myapp
    volumes:
      - postgres_data:/var/lib/postgresql/data
    ports:
      - "5432:5432"

volumes:
  postgres_data:

Comandos Docker Compose:

# Iniciar todos os serviços
docker-compose up -d

# Ver logs
docker-compose logs -f

# Parar serviços
docker-compose down

# Parar e remover volumes
docker-compose down -v

# Reconstruir imagens
docker-compose up -d --build

📝 Exercício Prático

Crie um ambiente Docker para uma aplicação Node.js com MongoDB:

  • Crie um Dockerfile otimizado para a aplicação
  • Crie um docker-compose.yml com Node.js e MongoDB
  • Use variáveis de ambiente para configurações
  • Configure volumes para persistência de dados
  • Teste a aplicação localmente
🎯 Solução de referência: 
# Dockerfile
FROM node:18-alpine
WORKDIR /app
COPY package*.json ./
RUN npm ci
COPY . .
EXPOSE 3000
CMD ["node", "server.js"]

# docker-compose.yml
version: '3.8'
services:
  app:
    build: .
    ports:
      - "3000:3000"
    environment:
      - MONGODB_URI=mongodb://mongo:27017/myapp
    depends_on:
      - mongo
  mongo:
    image: mongo:6
    volumes:
      - mongo_data:/data/db
volumes:
  mongo_data:
