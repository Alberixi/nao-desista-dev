⚡ JavaScript Assíncrono: Promises e Async/Await

📖 20 min de leitura 🎯 Nível: Avançado 💻 Código prático

📌 Introdução

JavaScript é single-threaded, mas isso não significa que não possamos executar operações assíncronas! Neste tutorial, vamos dominar Promises, Async/Await, tratamento de erros e como evitar o temido callback hell.

Entender programação assíncrona é essencial para trabalhar com APIs, bancos de dados, arquivos e qualquer operação que demande tempo.

🎯 Objetivos

  • Compreender o que é assincronicidade no JavaScript
  • Dominar Promises e seus métodos (then, catch, finally)
  • Aprender Async/Await de forma prática
  • Tratar erros corretamente em operações assíncronas
  • Executar múltiplas operações em paralelo vs sequencial

⚠️ Erro Comum #1: Callback Hell

❌ Erro: Aninhamento excessivo de callbacks 
// Código difícil de ler e manter
getUserData(userId, (user) => {
    getPosts(user.id, (posts) => {
        getComments(posts[0].id, (comments) => {
            getAuthor(comments[0].authorId, (author) => {
                console.log(author.name);
                // Mais callbacks aninhados...
            });
        });
    });
});

Problema: Código espaguete, difícil de debugar, sem tratamento adequado de erros.

✅ Correção: Usar Promises encadeadas 
getUserData(userId)
    .then(user => getPosts(user.id))
    .then(posts => getComments(posts[0].id))
    .then(comments => getAuthor(comments[0].authorId))
    .then(author => console.log(author.name))
    .catch(error => console.error('Erro:', error));

⚠️ Erro Comum #2: Esquecer o await em funções assíncronas

❌ Erro: Chamar função assíncrona sem await 
async function fetchUserData() {
    const response = fetch('https://api.example.com/user'); // Esqueceu o await!
    const data = response.json(); // Isso retorna uma Promise, não os dados
    console.log(data); // Promise {<pending>}
    return data;
}

Problema: A função retorna uma Promise pendente, não os dados esperados.

✅ Correção: Adicionar await 
async function fetchUserData() {
    const response = await fetch('https://api.example.com/user');
    const data = await response.json();
    console.log(data); // Dados reais!
    return data;
}

⚠️ Erro Comum #3: Não tratar erros adequadamente

❌ Erro: Ignorar tratamento de erros 
async function getData() {
    const response = await fetch('https://api.invalida.com');
    const data = await response.json(); // Se a API falhar, o app quebra!
    return data;
}

Problema: Qualquer erro na requisição faz a aplicação parar de funcionar.

✅ Correção: Usar try/catch 
async function getData() {
    try {
        const response = await fetch('https://api.invalida.com');
        if (!response.ok) {
            throw new Error(`HTTP error! status: ${response.status}`);
        }
        const data = await response.json();
        return data;
    } catch (error) {
        console.error('Erro ao buscar dados:', error.message);
        // Retornar um valor padrão ou relançar o erro
        return null;
    }
}

⚠️ Erro Comum #4: Execução sequencial desnecessária

❌ Erro: Esperar uma requisição para começar outra 
async function fetchAll() {
    const user = await fetchUser(); // Espera 1s
    const posts = await fetchPosts(); // Espera mais 1s (já 2s total)
    const comments = await fetchComments(); // Espera mais 1s (3s total)
    return { user, posts, comments };
}

Problema: Operações independentes executam sequencialmente, desperdiçando tempo.

✅ Correção: Execução paralela com Promise.all 
async function fetchAll() {
    const [user, posts, comments] = await Promise.all([
        fetchUser(),
        fetchPosts(),
        fetchComments()
    ]);
    return { user, posts, comments };
}
// Tempo total: apenas 1 segundo (as mais lentas das três)
💡 Dica: Use Promise.allSettled() quando não quiser que uma falha interrompa as outras operações.

🚀 Evolução: Do callback ao Async/Await moderno

1. Callback (Código antigo)

function getUser(id, callback) {
    setTimeout(() => {
        callback({ id: id, name: 'João' });
    }, 1000);
}

getUser(1, (user) => {
    console.log(user);
});

2. Promise (Melhor, mas ainda verboso)

function getUser(id) {
    return new Promise((resolve, reject) => {
        setTimeout(() => {
            resolve({ id: id, name: 'João' });
        }, 1000);
    });
}

getUser(1)
    .then(user => console.log(user))
    .catch(error => console.error(error));

3. Async/Await (Moderno e elegante)

async function getUser(id) {
    // Simula operação assíncrona
    return new Promise((resolve) => {
        setTimeout(() => {
            resolve({ id: id, name: 'João' });
        }, 1000);
    });
}

async function displayUser() {
    try {
        const user = await getUser(1);
        console.log(user);
    } catch (error) {
        console.error('Erro:', error);
    }
}
🎯 Evolução:

Comece entendendo callbacks, depois migre para Promises e finalmente domine Async/Await. Cada etapa traz mais legibilidade e controle de erros!

💡 Dicas Avançadas

  • Promise.race(): Retorna a primeira Promise que resolver ou rejeitar
  • Promise.any(): Retorna a primeira Promise que resolver (ignora rejeições)
  • Top-level await: Em módulos ES, você pode usar await fora de funções async
  • AbortController: Cancelar requisições fetch em andamento
  • Debounce e Throttle: Controlar a taxa de chamadas de funções assíncronas

Exemplo com AbortController

const controller = new AbortController();
const signal = controller.signal;

fetch('https://api.example.com/data', { signal })
    .then(response => response.json())
    .catch(err => {
        if (err.name === 'AbortError') {
            console.log('Requisição cancelada');
        }
    });

// Cancelar a requisição após 3 segundos
setTimeout(() => controller.abort(), 3000);
