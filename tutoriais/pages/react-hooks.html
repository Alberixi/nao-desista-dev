⚛️ React Hooks: useMemo e useCallback

📖 18 min de leitura 🎯 Nível: Intermediário 💻 Otimização de performance

📌 Introdução

React é conhecido por sua performance, mas componentes mal otimizados podem causar renderizações desnecessárias e lentidão. useMemo e useCallback são hooks essenciais para memorizar valores e funções, evitando recálculos e re-renderizações desnecessárias.

Neste tutorial, vamos entender quando e como usar cada um deles, além de identificar erros comuns que prejudicam a performance.

🎯 Objetivos

  • Entender a diferença entre useMemo e useCallback
  • Identificar quando usar cada hook
  • Evitar renderizações desnecessárias em componentes filhos
  • Otimizar cálculos pesados com useMemo
  • Compreender o array de dependências

⚠️ Erro Comum #1: Cálculos pesados em toda renderização

❌ Erro: Calcular valores pesados sem memoização 
function ProductList({ products, filter }) {
    // Este cálculo roda em TODA renderização, mesmo se products/filter não mudaram
    const filteredProducts = products.filter(p => 
        p.name.toLowerCase().includes(filter.toLowerCase())
    ).sort((a, b) => a.price - b.price);
    
    return (
        <ul>
            {filteredProducts.map(p => <li key={p.id}>{p.name}</li>)}
        </ul>
    );
}

Problema: Se products tiver 1000+ itens, esse filtro e ordenação rodam a cada re-renderização, causando lentidão.

✅ Correção: Usar useMemo 
function ProductList({ products, filter }) {
    const filteredProducts = useMemo(() => {
        console.log('Calculando filtro...');
        return products
            .filter(p => p.name.toLowerCase().includes(filter.toLowerCase()))
            .sort((a, b) => a.price - b.price);
    }, [products, filter]); // Só recalcula quando products ou filter mudam
    
    return (
        <ul>
            {filteredProducts.map(p => <li key={p.id}>{p.name}</li>)}
        </ul>
    );
}

⚠️ Erro Comum #2: Funções recriadas a cada renderização

❌ Erro: Passar função nova para componente filho 
function Parent() {
    const [count, setCount] = useState(0);
    
    // Esta função é recriada em TODA renderização do Parent
    const handleClick = () => {
        console.log('Clicou!');
    };
    
    return (
        <div>
            <button onClick={() => setCount(c => c + 1)}>Count: {count}</button>
            <ExpensiveComponent onClick={handleClick} />
        </div>
    );
}

// ExpensiveComponent vai re-renderizar mesmo se nada mudou,
// porque a prop onClick é uma nova função a cada renderização

Problema: Componentes filhos otimizados com React.memo re-renderizam desnecessariamente porque a função é sempre uma nova referência.

✅ Correção: Usar useCallback 
function Parent() {
    const [count, setCount] = useState(0);
    
    // useCallback mantém a mesma referência da função
    const handleClick = useCallback(() => {
        console.log('Clicou!');
    }, []); // Array vazio = função nunca muda
    
    return (
        <div>
            <button onClick={() => setCount(c => c + 1)}>Count: {count}</button>
            <ExpensiveComponent onClick={handleClick} />
        </div>
    );
}

const ExpensiveComponent = React.memo(({ onClick }) => {
    console.log('Renderizou ExpensiveComponent');
    return <button onClick={onClick}>Clique</button>
});

⚠️ Erro Comum #3: Array de dependências vazio ou incorreto

❌ Erro: Dependências incompletas ou desnecessárias 
function SearchComponent({ searchTerm }) {
    // Erro: dependência vazia, mas usa searchTerm
    const results = useMemo(() => {
        return performExpensiveSearch(searchTerm);
    }, []); // Nunca atualiza quando searchTerm muda!
    
    // Erro: dependência completa, mas a função nunca muda
    const handleSearch = useCallback(() => {
        console.log(searchTerm);
    }, [searchTerm, someOtherState, anotherState]); // Dependências demais
    
    return <div>...</div>
}

Problema: O useMemo nunca recalcula (dependências faltando) ou recalcula desnecessariamente (dependências demais).

✅ Correção: Dependências corretas 
function SearchComponent({ searchTerm }) {
    // Correto: inclui searchTerm como dependência
    const results = useMemo(() => {
        return performExpensiveSearch(searchTerm);
    }, [searchTerm]); // Recalcula apenas quando searchTerm muda
    
    // Correto: inclui apenas as dependências usadas
    const handleSearch = useCallback(() => {
        console.log(searchTerm);
    }, [searchTerm]); // Atualiza quando searchTerm muda
    
    return <div>...</div>
}

// Dica: use eslint-plugin-react-hooks para verificar dependências automaticamente

🚀 Evolução: Do componente simples à otimização completa

1. Componente sem otimização

function TodoList({ todos, filter }) {
    const filteredTodos = todos.filter(t => t.text.includes(filter));
    
    return (
        <ul>
            {filteredTodos.map(todo => <TodoItem key={todo.id} todo={todo} />)}
        </ul>
    );
}

2. Com useMemo e React.memo

const TodoItem = React.memo(({ todo }) => {
    return <li>{todo.text}</li>
});

function TodoList({ todos, filter }) {
    const filteredTodos = useMemo(() => {
        return todos.filter(t => t.text.includes(filter));
    }, [todos, filter]);
    
    return (
        <ul>
            {filteredTodos.map(todo => <TodoItem key={todo.id} todo={todo} />)}
        </ul>
    );
}

3. Otimização avançada com useCallback

const TodoItem = React.memo(({ todo, onToggle }) => {
    return (
        <li>
            <input 
                type="checkbox" 
                checked={todo.completed}
                onChange={() => onToggle(todo.id)}
            />
            {todo.text}
        </li>
    );
});

function TodoList({ todos, filter, onToggle }) {
    const filteredTodos = useMemo(() => {
        return todos.filter(t => t.text.includes(filter));
    }, [todos, filter]);
    
    const handleToggle = useCallback((id) => {
        onToggle(id);
    }, [onToggle]);
    
    return (
        <ul>
            {filteredTodos.map(todo => (
                <TodoItem 
                    key={todo.id} 
                    todo={todo} 
                    onToggle={handleToggle}
                />
            ))}
        </ul>
    );
}
🎯 Evolução:

Comece com componentes simples, adicione memoização em cálculos pesados, depois otimize callbacks e use React.memo para componentes que não precisam re-renderizar frequentemente.

💡 Dicas Avançadas

  • useMemo vs useCallback: useMemo memoriza o valor retornado, useCallback memoriza a função em si
  • useReducer: Alternativa ao useState para estados complexos, muitas vezes elimina necessidade de useCallback
  • React DevTools: Use a ferramenta "Highlight updates" para identificar componentes que re-renderizam desnecessariamente
  • Context API: Cuidado com Context que muda frequentemente, pode causar re-renderizações em massa

Exemplo de uso com useReducer

const initialState = { count: 0 };

function reducer(state, action) {
    switch (action.type) {
        case 'increment':
            return { count: state.count + 1 };
        case 'decrement':
            return { count: state.count - 1 };
        default:
            return state;
    }
}

function Counter() {
    const [state, dispatch] = useReducer(reducer, initialState);
    
    // dispatch é estável, não precisa de useCallback!
    return (
        <div>
            Count: {state.count}
            <button onClick={() => dispatch({ type: 'increment' })}>+</button>
        </div>
    );
}

📝 Exercício Prático

Crie um aplicativo de lista de tarefas com as seguintes características:

  • Lista com 1000+ itens
  • Filtro por texto (useMemo para filtrar)
  • Botão para marcar todos como concluídos (useCallback)
  • Contador de tarefas pendentes (useMemo)
  • Cada item deve ser um componente memoizado
🎯 Desafio extra: Meça a performance antes e depois das otimizações usando React DevTools ou console.time.
